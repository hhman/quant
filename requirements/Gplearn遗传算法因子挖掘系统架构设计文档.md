## 文档概述

本文档描述基于 Gplearn 遗传算法的自动化因子挖掘系统的完整架构设计。核心创新点在于通过**数据展平与还原机制**，使原本只支持二维数据的 Gplearn 框架能够处理三维面板数据，从而真正实现时间序列因子的自动发现。

### 核心设计思想

```
三维面板数据 → 展平为二维 → Gplearn 训练 → 还原为三维 → 计算三维适应度
```

**关键优势**：
- 无需修改 Gplearn 源码
- 支持时间序列算子（MA、STD、CORR 等）
- 支持截面算子（Rank、ZScore 等）
- 保留完整的自动发现价值

# 第一阶段：独立挖掘模块

## 1.1 模块定位与目标

### 核心定位

第一阶段是一个**完全独立的因子发现引擎**，通过数据展平技术将三维面板数据适配到 Gplearn 框架，实现时间序列因子的自动挖掘。该模块不依赖现有的因子评估体系（Step2-4），仅使用 Qlib 作为数据源。

### 设计目标

| 目标类别 | 具体目标 | 验收标准 |
|---------|---------|---------|
| **功能性** | 自动发现包含时间序列特征的因子表达式 | 输出 20-30 个候选表达式 |
| **独立性** | 不依赖 Step2-4 评估模块 | 可独立运行和测试 |
| **性能** | 在合理时间内完成挖掘 | 500 日数据 < 2 小时 |
| **质量** | 发现的因子具有预测能力 | 训练期 IC 均值 > 0.03 |

### 设计原则

1. **数据展平**：将三维面板数据展平为二维，适配 Gplearn 输入
2. **边界控制**：通过删除边界数据控制跨股票污染
3. **三维适应度**：在适应度函数中还原为三维数据计算
4. **低耦合**：与现有评估系统通过标准接口（表达式字符串）交互

### 核心机制详解

#### 机制 1：数据展平与还原

**核心思想**：将三维面板数据通过 MultiIndex 机制展平为二维数据，利用 Gplearn 训练后再还原为三维计算适应度。

**数据流转**：

```
原始数据:
  维度: (n_stocks, n_dates, n_features)
  结构: MultiIndex (instrument, datetime)

展平后:
  维度: (n_stocks × n_dates, n_features)
  结构: 扁平化数组
  特征: 每个样本是"某股票某日"的数据

还原后:
  维度: (n_dates, n_stocks)
  结构: DataFrame (index=日期, columns=股票)
  用途: 计算三维适应度
```

**关键点**：
- 展平时保留 MultiIndex，用于追踪样本身份
- 训练时 Gplearn 看到的是二维数据
- 适应度计算时通过 unstack 还原为三维

#### 机制 2：时间序列算子实现

**核心思想**：利用 Pandas Rolling 算子在展平数据上实现时间序列计算，通过边界控制处理跨股票污染。

**工作原理**：

```
展平数据示例:
  股票A: [10, 11, 12, 13, 14, ...]  (5日数据)
  股票B: [20, 21, 22, 23, 24, ...]  (5日数据)

展平后: [10, 11, 12, 13, 14, 20, 21, 22, 23, 24, ...]
         ↑ 股票A连续5日    ↑ 股票B连续5日

Rolling(3) 计算:
  结果: [NaN, NaN, 11, 12, 13, NaN, NaN, 21, 22, 23, ...]
  说明: 对每3个连续数据计算均值
```

**边界问题**：
- 在股票A和B的交界处会产生跨股票计算
- 例如: MA(13, 14, 20) 包含了A的最后2天和B的第1天

**解决方案**：
- 记录每只股票的起始位置（边界索引）
- 在适应度计算时删除边界后 N 天的数据（N = 最大窗口大小）

#### 机制 3：三维适应度计算

**核心思想**：在适应度函数内部将二维预测结果还原为三维面板数据，计算真正的截面时序适应度。

**计算流程**：

```
输入: y_pred_flat (n_stocks × n_dates,) - 二维预测结果

步骤 1: 边界处理
  - 删除每只股票前 N 天的数据（N = 窗口大小）
  - 目的: 去除跨股票污染的数据

步骤 2: 数据还原
  - 利用 MultiIndex 将 y_pred_flat 还原为面板
  - 方法: DataFrame.unstack('instrument')
  - 结果: (n_dates, n_stocks) DataFrame

步骤 3: 适应度计算
  - 按日期（行）计算截面 Rank IC
  - 方法: df.corrwith(y, axis=0, method='spearman')
  - 结果: 每日一个 IC 值

步骤 4: 聚合
  - 对所有日期的 IC 取平均
  - 结果: 单个标量适应度值

输出: 适应度值（越大越好）
```

**关键优势**：
- 支持真正的三维适应度（时间 × 截面）
- 可以计算时序稳定的指标（多日 IC 均值）
- 避免单日截面数据的偶然性

#### 机制 4：两套算子系统

**核心思想**：训练时使用适合展平数据的算子，评估时使用适合面板数据的算子，两者功能等价但实现不同。

**算子对比**：

| 算子类型 | 训练算子 | 评估算子 | 输入格式 |
|---------|---------|---------|---------|
| **移动平均** | Pandas Rolling on flatten | Pandas Rolling on panel | 1D array vs DataFrame |
| **标准差** | Pandas Rolling on flatten | Pandas Rolling on panel | 1D array vs DataFrame |
| **相关系数** | Pandas Rolling on flatten | Pandas Rolling on panel | 2 arrays vs 2 DataFrames |
| **截面排名** | N/A | DataFrame.rank() | 仅评估时使用 |
| **加减乘除** | Numpy ufuncs | Pandas ops | 一致 |

**实现策略**：
- 训练算子：输入为展平的一维数组，使用 Pandas Series 操作
- 评估算子：输入为面板数据，使用 Pandas DataFrame 操作
- 功能等价：确保相同的表达式在两个阶段产生相同结果

## 1.3 核心组件

### 组件 1：数据适配器

**职责**：将 Qlib 多维面板数据展平为 Gplearn 可接受的二维格式

**输入**：
- MultiIndex DataFrame（索引：instrument, datetime）
- 列：基础特征（$close, $volume, $total_mv 等）
- 列：目标变量（ret_1d, ret_1w, ret_1m）

**处理逻辑**：
1. 数据清洗：处理缺失值、异常值
2. 保持 MultiIndex：用于后续还原
3. 提取特征矩阵：转换为 NumPy 数组
4. 展平：自动展平为二维（n_samples, n_features）

**输出**：
- X_flat：二维数组 (n_stocks × n_dates, n_features)
- y_flat：一维数组 (n_stocks × n_dates,)
- index：MultiIndex（用于追踪样本身份）

**关键设计**：
- 保留完整的 MultiIndex 信息
- 记录每只股票的边界位置
- 支持增量加载（大数据集分块处理）

### 组件 2：时间序列算子库

**职责**：提供基于 Pandas Rolling 的时间序列算子

**算子分类**：

**趋势类算子**：
- SMA（Simple Moving Average）：简单移动平均
- EMA（Exponential Moving Average）：指数移动平均
- 支持多窗口：5日、10日、20日、60日

**动量类算子**：
- MOM（Momentum）：价格动量
- ROC（Rate of Change）：变化率
- DELTA：一阶差分

**波动率算子**：
- STD（Standard Deviation）：滚动标准差
- ATR（Average True Range）：真实波幅

**相关性算子**：
- CORR：滚动相关系数（两个序列）

**实现特点**：
- 使用 Pandas Series.rolling() 实现
- 通过 flatten() 处理输入数据
- 返回 NumPy 数组（Gplearn 要求）
- 处理 NaN 值（边界情况）

### 组件 3：截面算子库

**职责**：提供横截面统计算子

**算子类型**：

**排名算子**：
- Rank：横截面排名（归一化到 0-1）
- 支持多种排名方法（average, min, max, dense）

**标准化算子**：
- ZScore：横截面标准化（均值0，标准差1）

**分位数算子**：
- Quantile：分位数函数
- 中位数、四分位数等

**实现特点**：
- 仅在评估阶段使用
- 训练时不建议使用（需要在 Gplearn 内部实现分组）
- 可预计算为基础特征（如 Rank($close)）

### 组件 4：适应度函数引擎

**职责**：计算三维适应度，评估因子预测能力

**适应度类型**：

**截面 Rank IC**（默认）：
- 计算方法：每日截面 Spearman 相关系数
- 聚合方法：多日均值
- 优势：稳健，不受异常值影响

**时序 IC**：
- 计算方法：每只股票时序相关系数
- 聚合方法：多股票均值
- 用途：评估因子对个股的预测稳定性

**复合适应度**：
- 组合：IC + 稳定性惩罚
- 公式：IC - λ × std(IC)
- 目的：鼓励稳定的因子

**计算流程**：
1. 边界删除：去除跨股票污染
2. 数据还原：unstack 为面板
3. 截面计算：按日期计算相关系数
4. 聚合：多日均值/标准差
5. 返回：标量适应度值

### 组件 5：表达式输出

**职责**：直接输出 Gplearn 表达式，保留完整计算信息

**设计决策**：
- ❌ **移除表达式转换器**：不转换为 Qlib 语法
- ✅ **直接使用 Gplearn 表达式**：清晰可读的前缀表达式
- ✅ **由 LLM agent 按需转换**：需要时再转换为目标格式

**输出格式**：

```python
{
    'rank': 1,
    'expression': 'ts_rank(momentum(min(std(-0.021))))',  # Gplearn 格式
    'program': <_Program object>,                          # 可执行程序对象
    'fitness': 0.3772,                                    # 适应度分数
    'depth': 4,                                           # 表达式深度
    'length': 5,                                          # 节点数量
}
```

**优势**：
1. **简洁性**：避免复杂的 AST 解析和转换逻辑
2. **可读性**：Gplearn 前缀表达式本身就清晰易懂
3. **灵活性**：用户可以根据需要转换为任意格式
4. **维护性**：减少代码量，降低维护成本
5. **扩展性**：LLM agent 可以智能处理格式转换

**示例表达式**：
- 简单：`add(X0, X1)` → `$close + $volume`
- 复杂：`ts_rank(momentum(min(std(-0.021))))` → 时间序列排名(动量(最小值(标准差(-0.021))))
- 可直接理解：表达了清晰的因子逻辑

**如需转换为其他格式**：
- 由 LLM agent 处理
- 或编写独立的转换工具
- 或在应用层按需转换

## 1.6 关键技术决策

### 决策 1：数据展平策略

**选择：全量展平训练**

**理由**：
- 充分利用历史数据
- Gplearn 可以看到完整的时序模式
- 适应度计算基于多日数据，更稳健

**替代方案**：
- ❌ 逐日训练：失去时间连续性
- ❌ 随机采样：破坏时间序列结构

### 决策 2：边界污染控制

**选择：删除边界数据**

**理由**：
- 实现简单
- 效果可控
- 数据损失可接受（N 天 / 总天数）

**代价**：
- 每只股票损失 max_window 天数据
- 需要记录边界索引

**优化方向**：
- 支持多窗口算子时，按最大窗口删除
- 考虑滚动训练以减少数据损失

### 决策 3：适应度函数选择

**选择：截面 Rank IC（默认）**

**理由**：
- 稳健：不受异常值影响
- 时序连续：多日均值反映稳定性
- 行业标准：与 Step4 评估对应

**替代方案**：
- Pearson IC：受异常值影响
- 时序 IC：对个股预测，不是截面选择

### 决策 4：算子实现策略

**选择：Pandas Rolling 封装**

**理由**：
- 成熟稳定
- 支持复杂计算（如相关系数）
- 易于扩展

**限制**：
- 所有算子使用固定窗口（或有限窗口集）
- 需要处理 NaN 值

# 附录

## A. 核心创新点

### 创新 1：架构简化决策

**问题描述**：
- 是否需要将 Gplearn 表达式转换为 Qlib 语法？
- 转换逻辑复杂，维护成本高

**解决方案**：
- ❌ 移除表达式转换器
- ✅ 直接输出 Gplearn 表达式
- ✅ 由 LLM agent 按需转换

**优势**：
- 代码更简洁（减少 342 行转换器代码）
- 维护成本更低
- Gplearn 表达式本身清晰可读
- 灵活性更高（可转换为任意格式）

**示例**：
- Gplearn: `ts_rank(momentum(min(std(-0.021))))`
- 含义: 时间序列排名(动量(最小值(标准差(-0.021))))
- 清晰表达因子逻辑，无需转换

### 创新 2：数据展平方案

**问题描述**：
- Gplearn 仅支持二维输入
- 股票数据是三维面板（stocks × dates × features）

**解决方案**：
- 展平：三维 → 二维（保留 MultiIndex）
- 训练：Gplearn 使用二维数据
- 还原：适应度函数中还原为三维计算

**优势**：
- 无需修改 Gplearn 源码
- 保留完整的自动发现价值
- 支持时间序列算子

### 创新 3：时间序列算子实现

**问题描述**：
- Gplearn 算子是无状态的
- 时间序列算子需要历史信息

**解决方案**：
- 利用 Pandas Rolling 在展平数据上计算
- 通过边界删除控制跨股票污染
- 实现真正的时间序列算子

**示例算子**：
- SMA: 简单移动平均
- STD: 滚动标准差
- CORR: 滚动相关系数
- MOM: 动量

### 创新 4：三维适应度计算

**问题描述**：
- 传统适应度是单日截面
- 无法评估时序稳定性

**解决方案**：
- 在适应度函数中还原为三维面板
- 计算多日 Rank IC 均值
- 支持时序聚合指标

**优势**：
- 更稳健的适应度估计
- 避免单日偶然性
- 与 Step4 评估一致

## B. 术语表

| 术语 | 解释 |
|------|------|
| **数据展平** | 将三维面板数据展平为二维数据，每个样本是"某股票某日" |
| **边界污染** | 时间序列算子在股票交界处产生的跨股票计算 |
| **边界删除** | 删除每只股票前 N 天数据，去除污染 |
| **三维适应度** | 在面板数据上计算的适应度（时间 × 截面） |
| **MultiIndex** | Pandas 多层索引，用于追踪样本身份 |
| **unstack** | 将长格式数据转换为宽格式面板 |
| **Rank IC** | Spearman 相关系数，排序相关性 |
